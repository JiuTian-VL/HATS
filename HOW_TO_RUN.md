## Environment Setup

```bash
conda create -n hats python=3.10 -y
conda activate hats
pip install -r requirements.txt
cp .env.example .env
```

Fill in `.env` as needed:

```env
OPENAI_BASE_URL="https://api.openai.com/v1"
OPENAI_API_KEY="your_api_key"
OPENAI_API_MODEL="gpt-4o"
```

## Android Workflow

### 1) Launch the AVD

First create an AVD by following the official AndroidWorld documentation, then run:

```bash
python create_avd_batch_sh.py
bash run_avd.sh
```

### 2) Stage 1: Exploration

```bash
python create_mcts_batch_sh.py
bash run_mcts_batch.sh
```

### 3) Stage 2: Convert Exploration Trajectories to SFT Data

```bash
python all_execution_data_to_sft_data.py
```

### 4) Stage 3: Training

Train by following the official InternVL / Qwen training documentation.

### 5) Stage 4: Evaluation

First generate the merged document:

```bash
python fusion_all_documents.py
```

This generates `all_documents.pkl.zst`. Then follow [docs/androidworld_eval.md](docs/androidworld_eval.md) to integrate the HATS Agent into AndroidWorld and start evaluation.

## Web Workflow

### 1) Configure the WebArena Environment

First follow the AgentLab documentation:

- https://github.com/ServiceNow/AgentLab

Then complete the additional setup in [docs/webarena_setup.md](docs/webarena_setup.md), and fill in:

- `webarena_web/exp_env.sh`

### 2) Stage 1: Exploration

```bash
source webarena_web/exp_env.sh
python webarena_web/create_web_mcts_batch_sh.py
bash run_web_mcts_batch.sh
```

### 3) Stage 2: Convert Exploration Trajectories to SFT Data

```bash
python -m webarena_web.all_web_execution_data_to_sft_data
```

### 4) Stage 3: Training

Train by following the official InternVL / Qwen training documentation.

### 5) Stage 4: Evaluation

Run evaluation by following the AgentLab documentation. Recommended setting:

- `max_total_tokens=32768`

to avoid overly long HTML observations.

## Notes

- `run_avd.sh`, `run_mcts_batch.sh`, and `run_web_mcts_batch.sh` are generated by Python scripts. Run the corresponding Python scripts to create them.
- This workflow assumes a Linux + tmux environment.
